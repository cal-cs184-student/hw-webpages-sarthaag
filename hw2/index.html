Overview

This assignment entailed implementing various mesh operations with the halfedge data structure, including calculating the area-weighted vertex normal, performing edge flips, performing edge split, and applying Loop subdivision to refine and smooth meshes. A significant focus of these operations was ensuring that all modifications to the local mesh connectivity were preserved and maintained the correct halfedge data structure. From this assignment, we learned how useful and powerful the halfedge mesh representation can be for implementing local mesh edits and subdivision algorithms. In addition, it was intriguing to see how basic and simple local rules (e.g., Loop subdivision's rules) can lead to smooth surfaces being made from coarse, low-resolution meshes after multiple iterations of applying the same local rules to each triangle in the mesh.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

de Casteljau's algorithm begins with choosing an arbitrary number of points in a plane and connecting them together. Then, points are inserted on each edge through linear interpolation and those are connected together. This process repeats recursively until a final point is found that defines the Bezier curve. To implement this, we implemented functions that did one step of this algorithm in 2D and 3D, using for loops to iterate over all pairs of adjacent points and compute the linearly interpolated point between each pair. Then the last function recursively calls "evaluateStep" on the input points until only one is left, which is the one we're looking for.

Part 2: Bezier surfaces with separable 1D de Casteljau

de Casteljau algorithms extend to Bezier surfaces because the process is the same recursive interpolation used in Bezier curves but extended to two dimensions. For any point on the surface, the algorithm performs the 1D evaluation along one direction of the grid with parameter u, then again in the orthogonal direction with parameter v. In our implementation, we first loop through each row of control points and call evaluate1D with parameter u to compute an intermediate point for each row. These intermediate points are stored in a temporary vector. We then call evaluate1D again on this vector with parameter v, which evaluates the final Bézier curve and returns the point on the Bézier surface at (u, v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

Starting from the vertex’s half-edge, we get the coordinates of the three vertices of the corresponding triangle. We then compute two edge vectors from these points and compute the face normal by taking the cross product of these vectors. Since the magnitude of the cross product is proportional to twice the triangle’s area, each cross product weights each face normal by its area. After summing the normals for all faces by using twin()->next() to get to the next half-edge, we normalize the resulting vector to produce the unit vertex normal.

Part 5: Edge split

We implemented edge split by first collecting the local halfedge neighborhood around the selected edge and then separating the logic into two cases: interior edges and boundary edges. For an interior edge, we created a new midpoint vertex, three new edges, six new halfedges, and two new faces, then rewired the local connectivity so that the original two triangles were replaced by four smaller triangles. We updated the next, twin, vertex, edge, and face pointers for every halfedge in the affected region, and then reset representative halfedge pointers for all involved vertices, edges, and faces. We also assigned the new vertex’s position to the midpoint of the original edge and marked newly created elements appropriately for later subdivision steps. For the extra credit boundary case, we treated the boundary halfedge and interior halfedge separately. Instead of splitting two interior triangles, we only split the single non-boundary face and updated the virtual boundary loop by replacing the original boundary segment with two boundary halfedges meeting at the new midpoint vertex. This let the operation work correctly on meshes with open boundaries as well as closed meshes.

Part 6: Loop subdivision for mesh upsampling

We implemented loop subdivision in three steps. First, we computed the new positions for all original vertices and edges using the Loop subdivision rules and stored them in newPosition without modifying the mesh. Next, we split every original edge exactly once to create midpoint vertices and marked new vertices and edges appropriately. We then flipped each new non-boundary edge that connected one old vertex and one new vertex to obtain the correct subdivision connectivity. Finally, we updated each vertex’s position by copying newPosition into position.
